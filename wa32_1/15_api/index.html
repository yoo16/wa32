API とは

Web を利用していると「API」という文字を目にしたり、現場でもよく飛び交う言葉ではないでしょうか。

API（Application Programming Interface）は直訳すると、アプリケーションをプログラミングするためのインターフェースです。

インターフェイスとは

インターフェイス（Interface）はプログラミングだけに限らず利用される言葉で「操作をする手順・規則」といった意味があります。

  • ユーザインターフェイス
  • デバイスのインターフェイス
  • プログラミングのインターフェイス

などありますが、ソフトウェアの場合 API は外部アプリからアクセスできるよう設計された窓口です。

Web API

Web API は SNS、天気情報、交通情報といった無数のデータを、各会社や個人がインターネットを介してサービス提供しています。

例えば、Google Map の API は地図の経路や位置を取得するなど、Google が許す限り様々な処理ができます。

Google Maps Platform

Google Map Platform だけでも多種多様な API が存在するのがわかるでしょう。

API の利用制限を調べる

API を利用するには各サービスごとに、有料・無料や利用制限を確認する必要があります。

一般的な API は、サイトにアカウント登録して制限をかけるものが多いでが、中にはアカウント登録なしで無料で利用できるものもあります。 ただし、

各サービスによって回数制限をかけられるため、攻撃的なアクセスをしないよう注意が必要です。

Ajax

Ajax は「Asynchronous JavaScript XML」の略で、JavaScript と XML を使ってサーバと非同期通信する技術です。

非同期通信とは

クライアントとサーバのデータ通信において「同期」と「非同期通信」に分けられます。

「同期通信」はクライアントからリクエストしたらサーバからレスポンスがくるまで待ちます。 つまりレスポンスがくるまで、ユーザが操作できません。

「非同期通信」を利用すると、サーバからレスポンスがなくてもユーザ操作がすることができます。

XMLHttpRequest とは

JavaScript で Ajax を利用すると非同期通信でデータ送受信ができますが、そのリクエストに XMLHttpRequest を基本的に利用します。

XMLHttpRequest は、クライアントとサーバーと HTTP通信するためにブラウザに用意された API です。

Ajax のデータフォーマット

Ajax では XMLHttpRequest を利用してデータ通信しますが、クライアントとサーバーのデータ通信には決められたテキストデータフォーマットが必要です。

XML

そのデータフォーマットで利用されるのが「XML」というもので、 HTML 同様の DOM（Document Object Model）のツリー構造の言語です。 


    <?xml version="1.0" encoding="UTF-8" ?>
    <foods>
      <food>
        <name>Apple</name>
        <price>180</price>
      </food>

      <food>
        <name>Banana</name>
        <price>200</price>
      </food>
    </foods>

JSON

JSONは JavaScript Object Notation の略で、RFC 4627 に準拠した XML に代わるテキストデータフォーマットです。

Ajax のデータフォーマットは XML がデータ構造上大きく負荷がかかる傾向にあるため、「JSON」が多く利用されます。 


      {
        "foods": [
          {"name":"apple","price":180},
          {"name":"banana","price":150}
        ]
      }

また、JavaScript で利用できるデータフォーマットで開発されているため、ライブラリは必要ありません。 他の言語では JSON を扱うためのライブラリが必要です。

jQuery で Ajax を利用する

jQuery で Ajax 処理は色々ありますが古くからは $.ajax() を利用します。


      $.ajax({
      });

ちなみに、JavaScript ネイティブでは XMLHttpRequest クラスを利用します。


      let request = new XMLHttpRequest();

$.ajax() のパラメータはオブジェクトで指定します。 パラメータはいくつかありますが、よく利用するのは以下の通りです。 


      $.ajax({	
        url: 通信先のURL
        type: HTTPメソッド(post, get)
        data: 送信するデータ
        dataType: データタイプ（text, json, jsonp ...）
      }

CORS

CORS (Cross-Origin Resource Sharing) は、クロスドメイン（異なるドメイン）間で通信を共有する仕組みです。

クロスドメイン間の通信を安易に許可してしまうと、悪意のあるプログラムから攻撃対象となるためデフォルトでは Ajax通信はできません。

Origin とは

Origin は ドメインに「プロトコル」「ポート番号」を追加したものです。 


        Origin: http://yahoo.co.jp:80

ある Origin サーバーアプリから、別の Origin サーバーへのリクエストを許可することで、クロスドメイン間での Ajax 通信が実現できます。

JSONP

クロスドメイン間の通信が確立されても、JavaScript（jQuery）ではそもままデータ通信ができません。

JSONP（JSON with Padding）は、JavaScript 上でクロスドメイン制限を超えたデータ通信を実現します。

実際には別ドメインに対して JavaScript の実行権限をあたえる行為なので、信頼できるドメインに対して利用する必要があります。
